12.3. Tipos de campo predefinidos

Symfony incluye de serie muchos tipos de campos de formulario que cubren prácticamente todas las necesidades de la mayoría de aplicaciones web:

  • Campos de texto
    • text
    • textarea
    • email
    • integer
    • money
    • number
    • password
    • percent
    • search
    • url
  • Campos de selección
    • choice
    • entity
    • country
    • language
    • locale
    • timezone
  • Campos de fecha y hora
    • date
    • datetime
    • time
    • birthday
  • Otros campos
    • checkbox
    • file
    • radio
  • Campos para grupos
    • collection
    • repeated
  • Campos ocultos
    • hidden
    • csrf
  • Campos base
    • field
    • form

También puedes crear tus propios tipos de campo personalizados.

12.3.1. Configurando los campos de formulario

Cada tipo de campo tiene una serie de opciones que puedes utilizar para configurarlo. El campo dueDate por ejemplo se está mostrando como tres listas desplegables, pero puedes configurarlo para que sea una simple caja de texto (para que el usuario introduzca la fecha como una cadena de texto):

->add('dueDate', 'date', array('widget' => 'single_text'))

Figura 12.2 Campo de fecha mostrado como un cuadro de texto simple

Cada tipo de campo tiene diferentes opciones de configuración. Muchas de ellas son específicas para el tipo de campo y puedes encontrar los detalles en la documentación de cada tipo.

La opción required La opción de configuración más común es la opción required, que puedes aplicar a cualquier campo. El valor por defecto de required es true, lo que significa que los navegadores que soportan HTML5 aplicarán la validación que impide que el campo se deje vacío. Si no deseas este comportamiento, establece la opción required en tu campo a false o desactiva la validación de HTML5 tal y como se explicó en las secciones anteriores.

Ten en cuenta también que al establecer la opción required a true no estás validando la información en el lado del servidor. En otras palabras, si un usuario envía ese campo vacío (ya sea por ejemplo con un navegador antiguo o mediante un servicio web), será aceptado como un valor válido a menos que utilices la restricción NotBlank o NotNull de Symfony sobre la propiedad del objeto.

En otras palabras, la opción required es útil pero siempre se debe utilizar junto con la validación del lado del servidor.

La opción label El título de cada campo del formulario se establece mediante la opción label, que se puede aplicar individualmente a cada campo:

->add('dueDate', 'date', array(
    'widget' => 'single_text',
    'label'  => 'Due Date',
))

El título de un campo también se puede modificar en la propia plantilla, tal y como se explicará más adelante.
