Symfony2 contar intentos fallidos de login

Saludos. He seguido los manuales y he logrado realizar una aplicación con usuarios y roles en Symfony2. Mi controlador tiene lo siguiente para la función de login:

public function loginAction()
{
    $request = $this->getRequest();
    $session = $request->getSession();
 
    // obtiene el error de inicio de sesión si lo hay
    if ($request->attributes->has(SecurityContext::AUTHENTICATION_ERROR)) {
        $error = $request->attributes->get(SecurityContext::AUTHENTICATION_ERROR);
    } else {
        $error = $session->get(SecurityContext::AUTHENTICATION_ERROR);
    }
 
    return $this->render('CoacRiobambaAccesoBundle:Security:login.html.twig', array(
        // el último nombre de usuario ingresado por el usuario
        'last_username' => $session->get(SecurityContext::LAST_USERNAME),
        'error'         => $error,
    ));
}

Ahora necesito contabilizar el numero de intentos fallidos, es decir, que cada vez que ingrese la contraseña mal aumente un contador en la base de datos, para que después de 3 intentos, la cuenta se bloquee. El problema es que no se cómo o dónde tomar el error de contraseñas para proceder a cambiar el contador en la base de datos.

#1

@Geovana_Ch siempre que quieras hacer cosas avanzadas relacionadas con la seguridad y los usuarios en Symfony2, te recomiendo que eches un vistazo al código fuente del bundle FOSUserBundle, ya que seguramente sus creadores se han encontrado con los mismos problemas que tienes que resolver en tu aplicación.

Lamentablemente, no veo que el bundle FOSUserBundle soporte esta opción de limitar el número de intentos de login fallidos. Sin embargo, sí que tiene una característica cuyo funcionamiento es muy similar a lo que necesitas: cada vez que un usuario hace login, se guarda la fecha y la hora en la base de datos (tu caso es un poco más complicado, ya que si el login es incorrecto, tienes que incrementar el número de intentos del usuario y si el login es correcto, simplemente reseteas a 0 el número de intentos).

Observa el código de la clase LastLoginListener para ver cómo lo hacen. De forma resumida, tendrás que crear una clase de tipo event listener y tendrás que definir un servicio asociado al que le pasas como parámetro el servicio necesario para acceder a la base de datos.

Al margen del FOSUserBundle, existe otro bundle para Symfony2 llamado CCDNUserSecurityBundle y que hace algo muy parecido a lo que estás buscando. De nuevo te recomiendo que investigues su código fuente porque te puede servir de inspiración. En cualquier caso, creo que este bundle hace más cosas de las que necesitas, ya que para cada intento de login, guarda el usuario, la fecha/hora y la dirección IP desde la que se realizó ese intento.

Por último, recuerda que Symfony2 no sólo tiene una clase UserInterface, sino que existe otra más avanzada llamada AdvancedUserInterface y que añade propiedades al objeto del usuario que te servirán por ejemplo para marcar una cuenta como bloqueada (por haber superado el número máximo de intentos de login).
